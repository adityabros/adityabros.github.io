Projects

1.Maritime Visual

It is one of the most interesting project on which I have worked on. It is related to maps. Objective of this project was to visualize the whole marine activity which is going on around the world whether it's Wind, pressure, Well, Wave height and various weather factors and the Ships that are enrolled in our system.

Getting all the data which is required by a mariner at once place was a big relief for the clients of our product as the only thing which he needed to see was the page of this module as then he was able to visualize the whole activity of his fleet along with Historical(past) tracks of his vessel and also the future paths which were recommended by BOSS (Core Product of the firm for which I work ).

This project has solved the problem of many mariners as they got all the naval stuff they needed at once place.

The main power which we used to develop this awesome product is Leaflet Maps which is an Open Source Map library which is competitive to google maps though they are free but they provide almost all features as a map instead of being free which is also supported by the Open source community.

2.MIRROR

As the name suggest, this project does a similar kind of job, it mirrorizes the data from the ship into our servers. The need of the system arose as Ship masters used to send various kind of marine data using Excel Sheets and it took a good amount of data and as they have limitations on ship.

Mirror paved a way for masters to reduce their data consumption by 90 % as it only required an email to be sent from their system into our servers and the job was done

It is an Offline system system which only requires internet connection to transmit data and whole process is offline from the Ship side.

It consists of a well programmed HTML file that has latest capabilities and visuals which is also fascinating for the ship captain as earlier the excel sheet also looked boring.All types of reports for which there were different excel sheets earlier are now in a single file

After generating data , Ship master needs to send data through an email and our mails are automatically read by our programs which parses the email and scans the mail for ambiguity and submits the data into system.

4.Suspicious User Detector

Spam protection is something that every system should has , as nobody knows what could happen next and similar thing happened at our firm when we were in a similar kind of situation

A lot of trafic was coming from a single IP address and it was requesting a very cruical page which is the heaviest in our system and does a lot of calculation. It was not causing an Issue but was quite normal and similar attacks from multiple servers could have caused something

So I designed my own approach to solve this problem using Leaky Bucket Algorithm to log out a user that seemed abnormal but before that he was warned too ! just to confirm if he is a genuine person or a automatic bot doing the job of a spammer
